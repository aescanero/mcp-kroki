1. Get the application URL by running these commands:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "mcp-kroki.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if contains "LoadBalancer" .Values.service.type }}
     NOTE: It may take a few minutes for the LoadBalancer IP to be available.
           You can watch the status of by running 'kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "mcp-kroki.fullname" . }}'
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "mcp-kroki.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo http://$SERVICE_IP:{{ .Values.service.port }}
{{- else if contains "ClusterIP" .Values.service.type }}
  export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/name={{ include "mcp-kroki.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}")
  export CONTAINER_PORT=$(kubectl get pod --namespace {{ .Release.Namespace }} $POD_NAME -o jsonpath="{.spec.containers[0].ports[0].containerPort}")
  echo "Visit http://127.0.0.1:8084 to use your application"
  kubectl --namespace {{ .Release.Namespace }} port-forward $POD_NAME 8084:$CONTAINER_PORT
{{- end }}

2. Check the health of your MCP Kroki server:
{{- if contains "ClusterIP" .Values.service.type }}
  kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "mcp-kroki.fullname" . }} 8084:{{ .Values.service.port }}
  curl http://localhost:8084/health
{{- end }}

3. MCP Kroki Configuration:
{{- if .Values.kroki.enabled }}
   - Kroki server is deployed alongside MCP Kroki
   - Kroki URL: {{ include "mcp-kroki.krokiUrl" . }}
{{- else }}
   - Using external Kroki server: {{ .Values.config.krokiUrl }}

4. Supported diagram types:
   - Block Diagram Family: blockdiag, seqdiag, actdiag, nwdiag, packetdiag, rackdiag
   - General Purpose: bpmn, bytefield, c4plantuml, d2, dbml, ditaa, erd, excalidraw
   - Code & Architecture: graphviz, mermaid, nomnoml, pikchr, plantuml, structurizr
   - Specialized: svgbob, symbolator, umlet, vega, vegalite, wavedrom, wireviz
